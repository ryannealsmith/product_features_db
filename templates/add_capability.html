Capability Guidelines
What is a Capability?
  • A skill or ability that makes up a product feature
  • Implemented by one or more technical functions
  • Has clear, measurable success criteria
  • Owned by a parent product feature
Best Practices
  • Use descriptive, action-oriented names
  • Define specific, measurable success criteria
  • Set realistic timeline expectations
  • Include performance metrics where applicable
Dependencies
  • Select technical functions this capability depends on
  • Dependencies help with development sequencing
  • Consider foundational algorithms and interfaces
  • Think about what must exist before this capability works
